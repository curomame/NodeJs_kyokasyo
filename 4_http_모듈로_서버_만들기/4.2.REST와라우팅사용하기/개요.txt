서버에 요청을 보낼 때는 주소를 통해 내용을 표현
index.html 이면 index.html 보여달라는거임

요청의 내용이 주소를 통해 표현되므로 서버가 이해하기 쉬운 주소를 사용하는 것이 좋음

REST REpresentational State Transfer 서버의 자원을 정의하고 자원에 대한 주소를 지정하는 약속
다양한 규칙들이 있는데 이왕이면 잘 지켜서 해야함

주소의 의미는 명확히 전달하기 위해 명사로 구성됨
/user 면 사용자 정보에 관련된 자원을 요청
/post면 게시글에 관련된 자원을 요청하는 것

단순히 명사만 있으면 동작 이해하기 어려우느 요청 메서드라는 것을 활용
폼 데이터 전송시 get post 메서드 지정하는것과 같은 맥락

get - 서버에서 데이터 가져옴. 요청의 본문에 데이터 안넣음. 보내야하면 쿼리스트링 이용
post - 서버에 자원을 새로 등록, 요청의 본문에 새로 등록할 데이터를 넣어 보냄
put - 서버의 자원을 요청에 들어 있는 자원으로 치환, 요청의 본문에 치환할 데이터를 넣어 보냄
patch - 서버 자원의 일부만 수정, 요청의 본문에 일부 수정할 데이터를 넣어 보냄
delete - 서버 자원을 삭제, 본문에 데이터 안넣음
option - 요청 전 통신 옵션 설명 => 이후에 자세한 설명

주소 하나가 요청 메서드 여러개 가질 수 있음
애매하면 post
